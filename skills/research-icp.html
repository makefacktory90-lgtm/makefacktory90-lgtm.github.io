Автор: Ира Буян (@ira_and_ai) + Клод · Апрель 2026
Вдохновлено статьёй на Хабре про «Метод Франкенштейна» — там 2 часа, здесь 15 минут.

Что делает: Собирает портрет целевой аудитории через несколько нейросетей параллельно + локальные источники. В одном окне. Без ручной пересадки между вкладками.

СКАЧАТЬ .MD ВСЕ СКИЛЛЫ

Триггеры

  • «исследование ца», «анализ аудитории», «портрет клиента»
  • «исследуй ца для [продукт]»
  • «кто моя аудитория», «кому это продавать»

Принцип

«Метод Франкенштейна» в одном окне. Несколько моделей отвечают на один вопрос, каждая сильна в своём. Claude Code собирает всё в финальный отчёт.

Бюджет: через OpenRouter одно исследование ЦА обходится в копейки.

РольИсточникЗа что отвечает
Глубина + цитатымодель через OpenRouterанализ, нюансы, цитаты
Второе мнениемодель через OpenRouterсвежий взгляд, другой трейн
Цифры + фактыGeminiбольшой контекст, источники
Контрольный голосChatGPTподтверждение/опровержение
Синтез + отчётClaude (ваша подписка)структура, эмпатия, язык

Плюс локальные источники: Telegram-экспорты, интервью, опросы. Читаются в Claude Code напрямую.

Требования

Обязательно: Claude Code + MCP для веб-поиска

Желательно: PAL MCP + OpenRouter (для полного «Метода Франкенштейна» с несколькими моделями). Без них скилл работает, но чуть беднее.

Процесс

Шаг 0: Сбор входных данных

Claude спросит один раз:

  1. Продукт / услуга — что исследуем?
  2. Цель — позиционирование / новый продукт / лендинг / рассылка?
  3. Что уже есть? — TG-экспорты JSON, интервью, опросы, ссылки на конкурентов

Шаг 1: Метапромптинг (1 мин)

Claude сам пишет оптимальный промпт для исследования. Модель улучшает промпт на 30-40%.

«Ты — senior research strategist. Напиши оптимальный промпт для исследования ЦА продукта [X] с целью [Y]...»

Шаг 2: Параллельный запрос — субагенты

Субагент 1 — Claude (структура и портреты)

3-5 портретов ЦА (ФИО условное, возраст, роль, день из жизни)
Для каждого: 3 боли, 3 возражения, 3 триггера покупки
Язык аудитории: 10 фраз
Анти-аудитория: кому НЕ нужен

Субагент 2 — ChatGPT Deep Research (через OpenAI MCP)

Найти реальные цитаты, посты, обсуждения аудитории
(форумы, reddit, TG-каналы, отзывы)
15-20 цитат дословно + источник + контекст

Субагент 3 — Gemini (через Gemini MCP)

Цифры, статистика, размер рынка, тренды
10-15 фактов с цифрами + ссылка на источник
Не верь памяти — ищи через поиск

Шаг 3: Обработка закрытых источников

Если есть TG-экспорт JSON, интервью, опросы — 4-й субагент анализирует их локально (не загружает в сторонние сервисы). Это заменяет NotebookLM.

Шаг 4: Синтез (2-3 мин)

  1. Сверка противоречий — где модели не согласны? (часто самый сильный инсайт)
  2. Приоритизация болей — какие повторяются у 2+ источников?
  3. Язык аудитории — топ-20 фраз
  4. Финальный отчёт

3 уровня использования

L1 — Ручной (любой чат с AI)

Claude.ai / ChatGPT / Gemini — работают как есть. Открой скилл, скопируй промпты вручную в 3 окна, сравни результаты. Время: 2-4 часа.

L2 — Облегчённый (Claude Code)

Скачай скилл → положи в ~/.claude/skills/ → запусти. Один «голос» (Claude), но автоматизация. Время: ~25 минут.

L3 — Полный (Claude Code + PAL MCP + OpenRouter)

Несколько моделей параллельно, синтез за 15 минут. Настройка один раз ~20 минут + $5 на OpenRouter хватит на 2-3 месяца. Установи PAL MCP, впиши ключ, перезапусти Claude Code.

Правила

  1. Всегда метапромптинг на Шаге 1.
  2. Параллельно, не последовательно. Экономия x3.
  3. Локальные данные — локально. Не загружай TG-экспорты в сторонние сервисы.
  4. Не верить цифрам из памяти любой модели. Требовать источник.
  5. Противоречия ≠ ошибки. Там часто инсайт.
  6. 15 минут — пол исследования. Второй подход через день находит ещё слой.

⚠️ Проверка данных

Модели галлюцинируют на цифрах. Перед использованием в лендинге / продающем тексте — проверь: цифры, имена, кейсы, ссылки. Чужая цифра без проверки = твоя репутация при ошибке.

Troubleshooting

  • «MCP не настроен» — скилл работает в урезанном режиме (только Claude). Результат ~70%.
  • «TG-экспорт слишком большой» — разбей на куски по 50 MB.
  • «Модели дали разное» — это фича, не баг. Проси Claude выделить расхождения.
  • «Отчёт общий» — входные слабые. Уточни продукт, добавь источники.
