Вдохновлено статьёй на Хабре про «Метод Франкенштейна» — там 2 часа, здесь 15 минут.

Что делает: Собирает портрет целевой аудитории через 3 нейросети параллельно + локальные источники. В одном окне. Без ручной пересадки между вкладками.

СКАЧАТЬ .MD ВСЕ СКИЛЛЫ

Триггеры

  • «исследование ца», «анализ аудитории», «портрет клиента»
  • «исследуй ца для [продукт]»
  • «кто моя аудитория», «кому это продавать»

Принцип

«Метод Франкенштейна» в одном окне. 3 модели отвечают на один вопрос, каждая сильна в своём. Claude Code собирает всё в финальный отчёт.

Бюджет: через OpenRouter одно исследование ЦА ~$0.05-0.08.

Требования

Обязательно: Claude Code + Exa MCP

Желательно: OpenAI MCP + Gemini MCP (для полного «Метода Франкенштейна»). Без них скилл работает, но чуть беднее.

Процесс

Шаг 0: Сбор входных данных

Claude спросит один раз:

  1. Продукт / услуга — что исследуем?
  2. Цель — позиционирование / новый продукт / лендинг / рассылка?
  3. Что уже есть? — TG-экспорты JSON, интервью, опросы, ссылки на конкурентов

Шаг 1: Метапромптинг (1 мин)

Claude сам пишет оптимальный промпт для исследования, перед тем как запускать субагенты. Вы смотрите, одобряете или корректируете.

«Ты — senior research strategist. Напиши оптимальный промпт для исследования ЦА продукта [X] с целью [Y]...»

Шаг 2: Параллельный запрос — 3 субагента

Субагент 1 — Claude (структура и портреты)

3-5 портретов ЦА (ФИО условное, возраст, роль, день из жизни)
Для каждого: 3 боли, 3 возражения, 3 триггера покупки
Язык аудитории: 10 фраз
Анти-аудитория: кому НЕ нужен

Субагент 2 — ChatGPT Deep Research (через OpenAI MCP)

Найти реальные цитаты, посты, обсуждения аудитории
(форумы, reddit, TG-каналы, отзывы)
15-20 цитат дословно + источник + контекст

Субагент 3 — Gemini (через Gemini MCP)

Цифры, статистика, размер рынка, тренды
10-15 фактов с цифрами + ссылка на источник
Не верь памяти — ищи через поиск

Шаг 3: Обработка закрытых источников

Если есть TG-экспорт JSON, интервью, опросы — 4-й субагент анализирует их локально (не загружает в сторонние сервисы). Это заменяет NotebookLM-хак из статьи.

Шаг 4: Синтез (2-3 мин)

Claude получает результаты от 3-4 субагентов:

  1. Сверка противоречий — где модели не согласны? (часто самый сильный инсайт)
  2. Приоритизация болей — какие повторяются у 2+ источников?
  3. Язык аудитории — топ-20 фраз
  4. Финальный отчёт

Правила

  1. Всегда метапромптинг на Шаге 1. Модель улучшает промпт на 30-40%.
  2. 3 модели параллельно, не последовательно. Экономия x3.
  3. Локальные данные — локально. Не загружай TG-экспорты в сторонние сервисы.
  4. Не верить цифрам из памяти любой модели. Требовать источник.
  5. Противоречия ≠ ошибки. Там часто инсайт.
  6. 15 минут — пол исследования. Второй подход через день часто находит ещё слой.

⚠️ Проверка данных перед публикацией

Модели иногда галлюцинируют — особенно на цифрах. Перед использованием в лендинге / продающем тексте — проверь: конкретные цифры, имена, кейсы, ссылки. Чужая цифра без проверки = твоя репутация при ошибке.

Рекомендации ≠ решения

Скилл возвращает углы зрения и гипотезы, не готовые решения. Финальное решение — за тобой. У тебя есть контекст жизни, интуиция и ответственность за результат.

Troubleshooting

  • «MCP не настроен» — скилл работает в урезанном режиме (Claude + Exa). Результат ~70%.
  • «TG-экспорт слишком большой» — разбей на куски по 50 MB.
  • «3 модели дали разное» — это фича, не баг. Проси Claude выделить расхождения.
  • «Отчёт получился общим» — входные данные слабые. Уточни продукт, добавь источники.
