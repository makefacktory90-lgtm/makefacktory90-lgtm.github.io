Тема 01 // 05:36

ОТ ПРОТОТИПА ДО 180,000 ЗВЁЗД: РОЖДЕНИЕ OPENCLAW

Peter Steinberger продал PSPDFKit (работал на 1 млрд устройств) после 13 лет, «ушёл на пенсию» — и тут же влюбился в AI-агентов. Ждал полгода, что лаборатории создадут AI-ассистента, — никто не сделал. Надоело ждать, собрал сам.

Прототип за час: соединил WhatsApp с Claude Code CLI. Сообщение приходит, AI отвечает — всё. Из часового эксперимента выросли 300,000 строк кода.

«Я хотел AI-ассистента с апреля 2025. Ждал полгода, что лаборатории сделают сами. Никто не сделал — я разозлился и собрал сам.»

Тема 02 // 22:19

САМОМОДИФИЦИРУЮЩИЙСЯ AI-АГЕНТ

Steinberger сделал ключевую вещь: дал агенту полное знание о себе — свой код, документацию, среду выполнения. Самомодификация — не план, а побочный эффект. Когда AI «осознаёт» свою структуру, он начинает — сам себя улучшать.

Момент прозрения: Peter случайно отправил аудио агенту. Агент сам определил формат (opus), нашёл ffmpeg, достал OpenAI API-ключ, транскрибировал через Curl. Ни одна из этих инструкций не была запрограммирована.

«Люди говорят о самомодифицирующемся софте. Я просто взял и построил его. Даже не планировал особо — оно просто случилось.»

Тема 03 // 1:01:14

КАК КОДИТЬ С AI-АГЕНТАМИ

Steinberger перестроил процесс: не читает скучный код, фокусируется на логике. Рутина — перекладывание данных из формата в формат — автоматизирована. Мышление стало главной работой.

Главный сдвиг: промпты важнее кода. Когда приходит pull request, Steinberger читает промпты, а не код. Языки программирования больше не важны — важно системное мышление.

«Вся рутина написания кода автоматизирована, я двигаюсь намного быстрее. Но думать приходится намного больше. Это ментально ещё более выматывающе.»

Тема 04 // 1:38:52

CODEX 5.3 VS OPUS 4.6: БИТВА МОДЕЛЕЙ

Codex 5.3: «Это как тот странный парень в углу, с которым не хочется разговаривать, — но он надёжный и делает работу». Много читает код, сухой стиль — идеален для автономных задач.

Opus 4.6: «Немного глупый, но очень смешной коллега». Быстрый, интерактивный, креативный — идеален для живой работы. Но более импульсивен.

Штайнбергер использует оба: задача определяет инструмент. Он и Цукерберг устроили «10-минутный спор, что лучше — Claude Code или Codex».

«Я, наверное, самая большая бесплатная реклама Codex. Но Claude Code я использую каждый день. У нас с Цукербергом был 10-минутный спор: что лучше — Claude Code или Codex?»

Тема 05 // 2:52:20

AI-АГЕНТЫ ЗАМЕНЯТ 80% ПРИЛОЖЕНИЙ

Steinberger утверждает: 80% приложений на телефоне станут ненужными, когда AI получит доступ к API. MyFitnessPal, списки задач, трекеры рейсов, шоппинг-приложения — всё это функции, которые AI-агент может выполнить через один интерфейс.

Ключевая мысль: приложения должны стать API, хотят они этого или нет. UI-слой становится самим агентом. Переход от «скачай приложение для каждой задачи» к «скажи агенту, что тебе нужно».

«Если подумать, эта штука может заменить 80% приложений на вашем телефоне. Это не "когда-нибудь в будущем" — это происходит прямо сейчас. Приложения должны стать API, хотят они этого или нет.»

Тема 06 // 3:00:57

ЗАМЕНИТ ЛИ AI ПРОГРАММИСТОВ?

Steinberger даёт нюансированный ответ: AI не уничтожит программистов, но радикально изменит, что значит «программировать». Написание кода автоматизируется — остаётся проектирование систем, вкус, архитектура. Код становится ремеслом.

Ключевое разделение: «строители», которые мыслят системами — выиграют, а «кодеры», которые мыслят синтаксисом — проиграют. Программирование станет «как вязание свитера» — личное и творческое, не профессиональное узкое место.

«Программирование станет как вязание свитера. Системное мышление, архитектура, вкус и понимание зависимостей — вот что по-настоящему важно, и теперь эти навыки легко переносятся в любую область.»

Агент самостоятельно определил формат аудио, нашёл ffmpeg, достал API-ключ и транскрибировал — без единой инструкции. AI «знает» о мире достаточно, чтобы творчески комбинировать решения.
Вопрос 03 из 05 Как Steinberger сравнивает Codex 5.3 и Opus 4.6?

ГЛОССАРИЙ

ТерминОпределение
OpenClawOpen-source фреймворк AI-агентов от Peter Steinberger. Самый быстрорастущий проект в истории GitHub (180K+ звёзд). Превращает AI-модели в персональных ассистентов через WhatsApp, Telegram, Discord.
AI-агентAI-система, способная автономно планировать, выполнять многошаговые задачи и принимать решения. В отличие от чат-бота, агент действует без постоянного руководства.
СамомодификацияСпособность AI изменять свой собственный исходный код. В OpenClaw возникла как побочный эффект: агенту дали полный контекст о себе — он начал себя улучшать.
Agentic EngineeringПрофессиональный подход к работе с AI-агентами. Термин Steinberger для отличия от «vibe coding». Включает проектирование промптов, систем и архитектуры.
Vibe CodingНаписание кода с AI на интуиции, без чёткого плана. По мнению Steinberger: «Я занимаюсь agentic engineering, а после 3 ночи перехожу на vibe coding — и потом жалею».
Soul.mdФайл конфигурации в OpenClaw, определяющий «личность» AI-агента. Вдохновлён Constitutional AI: задаёт границы, тон и поведение.
Prompt InjectionАтака на AI через вредоносные промпты, заставляющие агента выполнить нежелательные действия. По Steinberger: «Проблема не решена. Не используйте дешёвые модели — они наивные».
Codex 5.3Модель OpenAI для кодинга. По Steinberger: «Странный парень в углу, с которым не хочется разговаривать, — но он надёжный и делает работу».
Opus 4.6Модель Anthropic из семейства Claude. По Steinberger: «Немного глупый, но очень смешной коллега». Быстрый, креативный, интерактивный.
MCPModel Context Protocol — стандарт, позволяющий AI-агентам взаимодействовать с внешними сервисами и инструментами. Своего рода API для AI-мира.
